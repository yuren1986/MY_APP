# 端到端测试场景

## 场景一：完整产品管理流程

### 1.1 添加产品（扫码方式）
```
用户打开应用
  → 点击底部"添加"按钮
  → 弹出添加方式选择
  → 选择"扫码添加"
  → 进入扫码页面
  → 模拟扫码识别产品
  → 查看识别结果
  → 点击"保存"
  → 返回首页
  → 新产品出现在列表顶部
```

### 1.2 添加产品（手动方式）
```
用户打开应用
  → 点击底部"添加"按钮
  → 弹出添加方式选择
  → 选择"手动添加"
  → 填写产品名称
  → 选择分类
  → 选择到期日期
  → 选择存储方式
  → 点击"保存添加"
  → 添加成功提示
  → 新产品出现在列表
```

### 1.3 添加产品（语音方式）
```
用户打开应用
  → 点击底部"添加"按钮
  → 弹出添加方式选择
  → 选择"语音添加"
  → 弹出语音识别弹窗
  → 开始聆听
  → 说出产品信息
  → 识别结果回显
  → 确认或修改
  → 点击"确认添加"
  → 添加成功提示
```

## 场景二：产品详情管理

### 2.1 查看产品详情
```
用户打开首页
  → 点击产品卡片
  → 进入产品详情页
  → 查看产品基本信息
  → 查看保质期状态
  → 查看存储方式建议
  → 查看提醒设置
```

### 2.2 编辑产品信息
```
用户进入产品详情页
  → 点击右上角编辑图标
  → 进入编辑模式
  → 修改产品名称
  → 修改品牌
  → 修改到期日期
  → 点击"保存修改"
  → 返回查看模式
  → 验证信息已更新
```

### 2.3 修改提醒设置
```
用户进入产品详情页
  → 找到提醒设置区域
  → 点击提醒开关启用提醒
  → 选择提前提醒天数（如7天）
  → 设置自动保存
  → 提示"提醒设置已更新"
```

### 2.4 删除产品
```
用户进入产品详情页
  → 点击"删除此产品"按钮
  → 弹出确认对话框
  → 点击"确认删除"
  → 删除成功提示
  → 自动返回首页
  → 产品从列表中移除
```

## 场景三：提醒管理

### 3.1 查看提醒列表
```
用户点击底部"提醒"Tab
  → 进入提醒管理页面
  → 查看统计数据
  → 按紧急程度分组查看提醒
  → 紧急提醒优先显示
```

### 3.2 处理提醒
```
用户进入提醒管理页面
  → 找到需要处理的提醒
  → 点击提醒项
  → 进入产品详情
  → 进行相应操作
  → 返回提醒页面
  → 标记为已处理
```

### 3.3 删除提醒
```
用户进入提醒管理页面
  → 找到要删除的提醒
  → 点击删除按钮
  → 提醒从列表移除
```

## 场景四：用户信息管理

### 4.1 查看个人信息
```
用户点击底部"我的"Tab
  → 进入设置页面
  → 查看用户头像
  → 查看昵称
  → 查看性别
  → 查看手机号码
  → 查看个人简介
```

### 4.2 编辑个人信息
```
用户进入设置页面
  → 点击右上角编辑图标
  → 进入编辑模式
  → 修改昵称
  → 修改手机号码
  → 选择性别
  → 填写生日
  → 填写邮箱
  → 填写收货地址
  → 填写个人简介
  → 点击"保存"
  → 返回展示模式
  → 验证信息已更新
```

### 4.3 修改应用设置
```
用户进入设置页面
  → 找到"通用设置"区域
  → 切换深色模式开关
  → 切换大字号模式
  → 切换高对比度
  → 访问通知设置
```

## 场景五：数据持久化

### 5.1 添加后刷新页面
```
用户添加新产品
  → 返回首页
  → 新产品显示
  → 刷新页面
  → 产品仍然显示
```

### 5.2 编辑后数据保持
```
用户编辑产品信息
  → 保存修改
  → 返回首页
  → 再次进入详情页
  → 验证修改已保存
```

### 5.3 删除后数据移除
```
用户删除产品
  → 返回首页
  → 产品不在列表中
  → 刷新页面
  → 产品仍然不在列表
```

## 边界条件测试

### B1. 空产品列表
- 首次使用无数据时显示空状态提示
- 提供引导添加的提示信息

### B2. 产品已过期
- 过期产品显示红色标识
- 过期天数正确计算
- 可正常查看和删除

### B3. 搜索无结果
- 输入不存在的搜索词
- 显示"未找到相关物品"提示

### B4. 表单验证
- 添加产品时名称为空
- 显示错误提示
- 不允许提交

### B5. 网络/存储异常
- localStorage 不可用时
- 显示友好错误提示
- 不崩溃
